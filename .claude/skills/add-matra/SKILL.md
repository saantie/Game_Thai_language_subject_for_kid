# /add-matra

เพิ่มมาตราใหม่หรือคำใหม่ลง `src/data/matra.js` ให้ถูกต้องตาม schema
รัน `/validate-matra` โดยอัตโนมัติหลังเพิ่มเสร็จ

## อาร์กิวเมนต์ที่รับได้

- `/add-matra` — เพิ่มมาตราใหม่ทั้งชุด (wizard ถามทีละขั้น)
- `/add-matra <matra-id>` — เพิ่มคำใหม่ลงในมาตราที่มีอยู่แล้ว

---

## กรณี A: เพิ่มมาตราใหม่ทั้งชุด

### ขั้น 1 — รวบรวมข้อมูลก่อนแก้ไฟล์

ถามผู้ใช้ตามลำดับ (อย่าถามรวมกันในครั้งเดียว):

1. **id** (ภาษาอังกฤษ, kebab-case, ไม่ซ้ำกับที่มีอยู่ใน matra.js)
   ตรวจสอบว่าไม่ซ้ำก่อนรับ
2. **ชื่อมาตรา** (เช่น "แม่ กน", "แม่ กง")
3. **โหมด**: TWO_PART หรือ FILL_FINAL
   - ถ้า TWO_PART: ถามสระ (`sara`) และพยัญชนะต้น (`bubbles`)
   - ถ้า FILL_FINAL: ถามเสียงตัวสะกด (`finalSound`)
4. **คำ** — ถามทีละคำ รับ input แบบนี้:
   - FILL_FINAL: `display, lead, final, spell (คั่นด้วย /)` เช่น `ลิง, ลิ, ง, ลอ/อิ/งอ/ลิง`
   - TWO_PART: `display, lead` เช่น `กา, ก` (spell ใช้รูปแบบ: `lead ออกเสียง / อา / display`)
   
   สำหรับ FILL_FINAL ถามต่อว่า distractors คืออะไร (ตัวสะกดที่ผิด ≥2 ตัว คั่นด้วยเว้นวรรค)
   
   ถามว่าต้องการเพิ่มคำอีกไหม — วนซ้ำจนตอบว่าเสร็จแล้ว (ควรมี ≥3 คำ)

### ขั้น 2 — ตรวจ distractor ก่อนเขียน

สำหรับทุก FILL_FINAL word ตรวจว่า:
- `final` ไม่อยู่ใน `distractors`
- `distractors` ไม่ซ้ำกัน
- `spell` ตัวสุดท้าย = `display`

ถ้าพบปัญหา แจ้งและให้แก้ก่อนเขียนไฟล์

### ขั้น 3 — เขียนลง matra.js

เพิ่ม entry ใหม่ต่อท้าย `MATRA` array
ใช้ format เดียวกับ entry ที่มีอยู่ (2-space indent, trailing comma)

ตัวอย่าง output สำหรับ FILL_FINAL:
```js
{
  id: 'new-id',
  name: 'แม่ กน',
  mode: 'FILL_FINAL',
  finalSound: 'น',
  words: [
    { display: 'จาน', lead: 'จา', final: 'น', spell: ['จอ', 'อา', 'นอ', 'จาน'], distractors: ['ง', 'ม', 'ด'] },
  ],
},
```

### ขั้น 4 — validate

รัน `/validate-matra` ทันทีหลังเขียนไฟล์
ถ้า validate ผ่าน: รายงาน "✅ เพิ่มมาตรา [name] สำเร็จ"
ถ้าไม่ผ่าน: แสดง error และแก้ก่อน commit

---

## กรณี B: เพิ่มคำลงมาตราที่มีอยู่แล้ว `/add-matra <id>`

1. อ่าน matra.js หา entry ที่ `id` ตรง
2. แสดง mode และคำที่มีอยู่แล้ว
3. รับ input คำใหม่ (format เดียวกับขั้น A.1)
4. ตรวจว่า `display` ไม่ซ้ำกับที่มีอยู่
5. เพิ่มลงใน `words` array ของมาตรานั้น
6. รัน `/validate-matra`

---

## กฎห้ามทำ
- ห้าม hardcode คำ/ตัวสะกดใน `game.js` หรือไฟล์อื่น — ข้อมูลทั้งหมดต้องอยู่ใน `matra.js`
- ห้าม commit ถ้า `/validate-matra` ยังไม่ผ่าน
- ห้ามแก้ไขโค้ด game logic เพื่อรองรับโครงสร้างที่ต่างออกไป
- ห้ามเดา spelling ของคำภาษาไทย — ถ้าไม่แน่ใจให้ถามผู้ใช้
